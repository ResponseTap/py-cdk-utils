# Contributing

When contributing to this repository, please first discuss the change you wish to make via issue, email, or any other
method with the owners of this repository before making a change.

Please note we have a [code of conduct](CODE_OF_CONDUCT.md), please follow it in all your interactions with the project.

## Pull Request Process

1. Before you create a Pull Request, please ensure test coverage has been maintained and that all tests are passing.
1. This project uses [Semantic Versioning](https://semver.org/) based on the
   [Conventional Commit](https://www.conventionalcommits.org/en/v1.0.0/) commit message format. This is mandatory so
   please ensure your commit message conform.
1. Update the [README.md](README.md) to reflect any changes. Also update any other documentation which has changed as a
   result of this PR.
1. Pull Requests must be approved by a minimum of two authorised reviewers.
1. Once approved you should merge the Pull Request. If you are not authorised to do so, then ask the second of the two
   reviewers to do this on your behalf.
1. Once merged, the project will be versioning, packaged and published.
